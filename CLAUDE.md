## プロジェクト概要

クリーンアーキテクチャの原則に従った図書管理システムです。プロジェクトは現在、初期セットアップ段階にあります。

## アーキテクチャ

システムは以下のレイヤーを持つクリーンアーキテクチャに従います：

- **ドメイン層**: コアビジネスエンティティとルール（フレームワークに依存しない）
- **アプリケーション層**: ユースケースとアプリケーション固有のビジネスルール
- **インフラストラクチャ層**: 外部関心事（データベース、Webフレームワーク、外部サービス）
- **プレゼンテーション層**: ユーザーインターフェイスとAPIエンドポイント

## 技術スタック

- **フロントエンド**: React + TypeScript + Vite
- **スキーマバリデーション**: Zod
- **テスティング**: Vitest + Testing Library
- **ビルドツール**: Vite
- **パッケージマネージャー**: npm

## 開発コマンド

```bash
npm install      # 依存関係のインストール
npm run dev      # 開発サーバー起動
npm run build    # プロダクションビルド
npm run test     # テスト実行
npm run test:ui  # UIテストランナー起動
npm run lint     # ESLintによるコード検査
```

## 実装ガイドライン

図書システムに機能を追加する際：

- レイヤー間の厳密な分離を維持
- 依存関係は内側に向ける（外側のレイヤーが内側のレイヤーに依存）
- ドメインエンティティは外部レイヤーへの依存を持たない
- レイヤー間の疎結合のために依存性注入を使用

## 図書システムの一般的な機能

実装が期待される主要機能：

- 書籍管理（書籍の追加、削除、更新、検索）
- 会員管理（登録、プロフィール管理）
- 貸出操作（貸出、返却、予約）
- 延滞料金の計算
- 在庫追跡
- 検索とフィルタリング機能

## プロジェクト構造

```text
├── src/
│   ├── schemas/        # Zodスキーマ定義
│   ├── test/          # テスト設定
│   └── ...            # Reactコンポーネント
├── vite.config.ts     # Vite設定（Vitest設定を含む）
└── package.json       # 依存関係とスクリプト
```
